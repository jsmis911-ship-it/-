# Nebula Sakura Gesture Gallery

一个基于 Vite、React、TypeScript、Three.js 和 MediaPipe Hands 的沉浸式手势交互照片展示网页应用。

项目支持在浏览器本地上传最多 32 张图片，通过摄像头手势或页面按钮驱动 3D 粒子、星云漂浮、樱花树汇聚、电影感樱花雨、照片球爆散、中心大图展示和整理图库浏览。

## 功能特性

- 最多上传 32 张图片，图片仅在本机浏览器中处理，不上传服务器。
- 初始状态中，图片以多层 3D 空间关系漂浮在星空粒子环境里。
- 握拳后，粒子汇聚成稳定立体的幻想樱花树，照片分布在枝叶区域。
- 张开手掌后，樱花树散开形成自左向右的樱花雨，并进入照片展示流程。
- 照片切换使用区域停留触发：
  - 手掌停留在左侧区域：上一张。
  - 手掌停留在右侧区域：下一张。
  - 回到中心安全区短暂停留：解锁下一次切换。
- 张掌并停留在上方中央区域可重置循环。
- 支持无摄像头兜底：底部按钮和键盘快捷键可模拟主要手势。
- 移动端建议横屏使用，竖屏会提示切换方向。

## 技术栈

- Vite
- React
- TypeScript
- Three.js / WebGL
- MediaPipe Hands

## 本地运行

```bash
npm install
npm run dev
```

默认开发地址：

```text
http://127.0.0.1:5317
```

首次开启手势识别时，浏览器会请求摄像头权限。

## 构建

```bash
npm run build
```

构建产物会输出到 `dist/`，该目录不需要提交到仓库。

## 快捷键

- `F`：模拟握拳，汇聚成樱花树。
- `O` 或空格：模拟张掌，触发樱花雨和照片展示。
- `ArrowLeft`：上一张。
- `ArrowRight`：下一张。
- `ArrowUp` 或 `R`：重置循环。

## 隐私说明

- 上传的照片不会写入项目目录，也不会上传到服务器。
- 图片会在浏览器内降采样为 WebGL 预览纹理。
- 摄像头画面只用于本地手势识别，不会被保存。
- 仓库忽略了 `node_modules/`、`dist/`、测试截图、浏览器 profile、日志、`.env` 和常见本地图片文件，避免误提交个人信息。

## 项目结构

```text
src/
  components/       3D 场景、粒子、照片和樱花雨渲染
  hooks/            手势识别与区域停留交互
  lib/              图片降采样与纹理资源准备
  App.tsx           页面状态机和主交互流程
  styles.css        界面样式
public/
  mediapipe/        MediaPipe Hands 本地模型资源
```

## 注意事项

- 推荐使用桌面浏览器体验完整手势与 3D 效果。
- 上传大量 4K 图片会增加显存和内存占用，应用已做降采样处理。
- 如果摄像头不可用，可以使用按钮或键盘快捷键完整演示流程。
